# 다시서기 정신건강 평가관리 시스템 화면 구조 설계

## 1. 문서 목적

본 문서는 `00-project-overview.md`를 기준으로, 다시서기 정신건강 평가관리 시스템의 화면 구조를 실제 개발 가능한 수준으로 구체화한 설계 문서이다.  
이 문서는 기능 요구사항을 화면 단위로 분해하여, 프론트엔드 화면 설계, 라우팅 구성, 권한 노출 정책, 사용자 흐름 설계의 기준으로 사용한다.

---

## 2. 화면 구조 설계 원칙

### 2.1 기본 원칙
- 로그인 후 첫 화면은 **대상자 목록 화면**으로 한다.
- 본 시스템의 핵심 사용자 흐름은 **대상자 검색 → 대상자 선택 → 대상자 상세 → 검사 시작 → 척도 입력 → 세션 저장 → 세션 상세 확인**이다.
- 대상자 기본정보와 검사기록은 분리 관리하되, 화면에서는 대상자 중심 흐름이 자연스럽게 이어지도록 구성한다.
- 검사 결과는 **세션 단위로 저장**하되, 목록 화면에서는 실무 편의를 위해 **척도별 기록 단위** 조회를 지원한다.
- 삭제 기능은 제공하지 않고, **대상자는 오등록 처리**, **검사 세션/검사기록은 오입력 처리** 방식으로 상태를 변경한다.
- 팀 전체 열람이 가능한 구조를 기본으로 하되, 수정·관리 기능은 권한에 따라 제한한다.

### 2.2 권한 원칙
- **일반 사용자**: 대상자 및 검사기록 열람, 대상자 등록, 검사 수행, 통계 열람 가능
- **관리자**: 일반 사용자 기능 포함 + 회원가입 승인, 사용자 관리, 로그 확인, 백업 관리 가능
- **수정 가능 대상**: 작성자와 관리자만 가능
- **대상자 오등록 처리 가능 대상**: 작성자와 관리자만 가능
- **세션 오입력 처리 가능 대상**: 작성자와 관리자만 가능
- **오등록/오입력 항목 열람 가능 대상**: 작성자와 관리자만 가능

### 2.3 개인정보 및 민감정보 노출 원칙
- 대상자 연락처는 **대상자 상세 화면에서만 노출**한다.
- 대상자 연락처는 **팀 전체 열람 가능**으로 한다.
- 세션 참고 메모는 **입력 화면과 상세보기 화면에서만 확인 가능**하며, 팀 전체 열람 가능으로 한다.
- 세션 참고 메모는 **출력물에 포함하지 않는다.**

- 검사기록
- 통계

- 회원가입 승인
- 사용자 관리
- 로그 확인
- 백업 관리

### 4.3 메뉴 설계 원칙

- 별도 관리자 전용 시스템으로 분리하지 않는다.

## 5. 사용자 주요 흐름

### 5.1 대상자 검색 및 검사 수행 흐름
1. 로그인
2. 대상자 목록 진입
3. 이름 검색 + 필요 시 생년월일 필터 적용
4. 대상자 이름 클릭 또는 `상세보기` 버튼 클릭
5. 대상자 상세 화면 진입
6. `검사 시작` 버튼 클릭
7. 척도 선택 화면에서 필요한 척도 복수 선택
8. 척도 입력 화면에서 선택 척도를 시스템 고정 순서로 순차 입력
9. 세션 전체 요약 화면에서 내용 확인 및 메모 입력
10. 저장 후 세션 상세 화면으로 이동
11. 필요 시 출력 수행

### 5.2 대상자 신규 등록 흐름
1. 대상자 목록 화면 진입
2. 상단 `대상자 등록` 버튼 클릭
3. 대상자 등록 화면 진입
4. 기본정보 입력
5. 중복 가능 대상자 경고 확인
6. 등록 저장
7. 대상자 상세 화면으로 이동

### 5.3 기록 조회 흐름
1. 검사기록 목록 화면 진입
2. 날짜 범위 / 담당자 / 대상자명 / 오입력 포함 여부 필터 적용
3. 기록 목록에서 특정 척도 기록 선택
4. 해당 기록이 속한 세션 상세 화면 진입
5. 세션 전체 결과, 경고, 메모, 출력 기능 확인

## 6. 화면 상세 설계

## 6.1 로그인 화면

### 목적
- 사용자 인증을 수행하고 시스템에 진입한다.

### 진입 경로
- 시스템 첫 접속 시 진입
- 세션 만료 시 재진입

### 주요 구성 요소
- 아이디 입력 필드
- 비밀번호 입력 필드
- 로그인 버튼
- 회원가입 신청 이동 버튼
- 로그인 오류 메시지 영역

### 주요 동작
- 유효한 계정이면 로그인 성공 후 대상자 목록 화면으로 이동
- 승인 대기 계정은 로그인 불가 및 안내 메시지 표시
- 비밀번호 오류 시 오류 메시지 표시

---

## 6.2 회원가입 신청 화면

### 목적
- 신규 사용자가 계정 사용 승인을 요청한다.

### 주요 구성 요소
- 이름
- 아이디
- 비밀번호
- 연락처
- 직책 또는 역할
- 소속 팀
- 가입 신청 메모
- 신청 버튼

### 주요 동작
- 신청 완료 시 승인 대기 상태로 저장
- 관리자 승인 전까지 로그인 불가

### 비고
- 1인 1계정 원칙을 따른다.

---

## 6.3 대상자 목록 화면

### 목적
- 시스템의 메인 허브 화면으로서 대상자 조회, 검색, 신규 등록 진입을 제공한다.

### 기본 진입 화면 여부
- **로그인 후 기본 진입 화면**으로 사용한다.

### 주요 구성 요소
- 통합 검색창(이름 검색)
- 생년월일 보조 필터
- 대상자 등록 버튼
- 대상자 목록 테이블
- 페이지네이션 또는 목록 스크롤 영역

### 목록 기본 표시 항목
- 이름
- 생년월일
- 성별
- 사례번호
- 담당자
- 최근 검사일

### 상세 진입 방식
- 이름 클릭 또는 `상세보기` 버튼 클릭 시 대상자 상세 화면으로 이동
- 행 전체 클릭 방식은 사용하지 않는다.

### 주요 동작
- 이름 우선 검색, 필요 시 생년월일로 결과 축소
- 오등록 대상자는 기본 목록에서 제외
- 관리자 또는 작성자만 `오등록 포함` 옵션 사용 가능

---

## 6.4 대상자 등록 화면

### 목적
- 신규 대상자 기본정보를 등록한다.

### 주요 구성 요소
- 이름
- 성별
- 생년월일
- 연락처(선택 입력)
- 담당자 드롭다운
- 사례번호 안내 항목(읽기 전용)
- 저장 버튼
- 취소 버튼

### 입력 정책
- 사례번호는 사용자가 직접 입력하지 않는다.
- 사례번호는 저장 시 자동 생성된다.
- 담당자는 현재 로그인한 사용자를 기본값으로 자동 선택한다.
- 연락처는 선택 입력이다.

### 중복 처리 정책
- 동일한 이름 + 생년월일 대상자가 존재하면 등록 전 경고 메시지를 표시한다.
- 기존 대상자 후보를 보여주고, 필요 시 신규 등록을 계속 허용한다.

### 저장 후 이동
- 저장 완료 후 대상자 상세 화면으로 이동한다.

---

## 6.5 대상자 상세 화면

### 목적
- 대상자 단위 업무 허브 화면으로서 기본정보와 최근 검사 이력을 함께 제공한다.

### 주요 구성 요소
#### 상단 기본정보 영역
- 이름
- 생년월일
- 성별
- 사례번호
- 연락처
- 등록일
- 담당자

#### 상단 주요 버튼
- 검사 시작
- 정보 수정
- 전체 기록 보기

#### 하단 최근 검사기록 영역
- 최근 10건의 검사 세션 요약 목록
- 표시 항목 예시: 검사일시, 담당자, 시행 척도 수, 경고 여부

### 동작 규칙
- 연락처는 대상자 상세 화면에서만 노출한다.
- `정보 수정` 버튼은 작성자와 관리자에게만 노출한다.
- 최근 검사기록은 세션 단위로 표시한다.
- 세션 항목 클릭 시 세션 상세 화면으로 이동한다.

---

## 6.6 대상자 정보 수정 화면

### 목적
- 대상자 기본정보를 수정한다.

### 접근 권한
- 작성자와 관리자만 접근 가능

### 주요 구성 요소
- 이름
- 성별
- 생년월일
- 연락처
- 담당자
- 저장 버튼
- 취소 버튼

### 비고
- 사례번호는 수정 불가 항목으로 처리한다.

---

## 6.7 척도 선택 화면

### 목적
- 한 번의 검사 세션에서 시행할 척도를 선택한다.

### 주요 구성 요소
- 8개 척도 체크박스 목록
  - PHQ-9
  - GAD-7
  - mKPQ-16
  - K-MDQ
  - PSS-10
  - ISI-K
  - AUDIT-K
  - IES-R
- 검사 시작 버튼
- 취소 버튼

### 선택 정책
- 여러 척도 복수 선택 가능
- 추천 묶음 버튼은 초기 버전에 포함하지 않는다.
- 실제 진행 순서는 사용자가 선택한 척도만 대상으로 하되, 아래의 시스템 고정 순서를 따른다.
  1. PHQ-9
  2. GAD-7
  3. mKPQ-16
  4. K-MDQ
  5. PSS-10
  6. ISI-K
  7. AUDIT-K
  8. IES-R

---

## 6.8 척도 입력 화면

### 목적
- 선택된 척도를 순차적으로 입력하고 자동 채점 및 경고를 확인한다.

### 화면 구성 방식
- 한 번에 한 척도씩 입력하는 단계형 화면으로 구성한다.

### 주요 구성 요소
- 현재 척도명
- 문항 목록
- 응답 선택 UI
- 현재 척도 총점 실시간 표시
- 현재 척도 판정 실시간 표시
- 경고 배너 또는 주의 문구
- 이전 버튼
- 다음 버튼
- 진행 상태 표시 영역

### 진행 상태 표시 방식
- 상단에 선택된 척도 전체 목록을 표시한다.
- 완료 / 진행 중 / 대기 상태를 구분하여 표시한다.
- 사용자는 전체 세션 흐름을 한눈에 파악할 수 있어야 한다.

### 입력 정책
- 문항 응답 즉시 총점 자동 산출
- 척도별 판정 자동 표시
- 고위험 또는 주의 응답 발생 시 즉시 경고 표시
- 전 문항 응답 완료 전에는 저장 불가
- 임시저장 기능은 제공하지 않음
- 최종 저장 전까지 이전 척도로 돌아가 수정 가능

### 단계 완료 흐름
- 각 척도 입력 완료 후 다음 척도로 이동
- 마지막 척도 완료 후 세션 전체 요약 화면으로 이동

---

## 6.9 세션 전체 요약 화면

### 목적
- 저장 전 세션 전체 결과를 최종 확인하고 참고 메모를 입력한다.

### 주요 구성 요소
- 대상자 기본정보 요약
- 검사일시
- 담당자
- 이번 세션에서 시행한 척도 목록
- 척도별 총점
- 척도별 판정
- 척도별 경고 여부
- 세션 참고 메모 입력란
- 저장 버튼
- 기록 상세보기로 이동 버튼(저장 후 유효)

### 동작 규칙
- 세션 참고 메모는 팀 전체 열람 가능 정보로 취급한다.
- 세션 참고 메모는 출력물에는 포함하지 않는다.

### 저장 후 이동
- 저장 완료 후 방금 저장한 세션 상세 화면으로 이동한다.

---

## 6.10 세션 상세 화면

### 목적
- 저장된 세션 전체 결과를 확인하고 출력 및 관리 작업을 수행한다.

### 진입 경로
- 세션 저장 직후 자동 이동
- 대상자 상세 화면의 최근 세션 목록 클릭
- 검사기록 목록 화면에서 특정 기록 클릭

### 주요 구성 요소
#### 상단 요약 영역
- 대상자 기본정보 요약
- 검사일시
- 담당자
- 세션 ID 또는 식별 정보

#### 척도 결과 영역
- 세션에 포함된 척도 목록
- 각 척도의 총점
- 각 척도의 판정
- 각 척도의 경고 정보
- 클릭하여 해당 척도 세부 내용 열람 가능

#### 메모 영역
- 세션 참고 메모 표시

#### 출력 영역
- 공식 문서형 출력 버튼

#### 관리 영역
- 오입력 처리 버튼

### 상세 표시 정책
- 검사기록 목록에서 특정 척도 기록을 클릭하더라도, 실제 진입은 해당 기록이 속한 **세션 전체 상세 화면**으로 한다.
- 이 경우 클릭한 척도는 기본 강조 표시한다.

### 출력 정책
- 공식 문서형 출력 기능은 세션 상세 화면에서만 제공한다.
- 출력 기준은 항상 세션 단위이다.

### 오입력 처리 정책
- 오입력 처리 버튼은 하단 관리 영역에만 배치한다.
- 작성자와 관리자에게만 노출한다.
- 실행 전 확인 팝업을 제공한다.

### 기록 수정 정책
- 저장된 검사기록은 직접 수정하지 않는다.
- 잘못 입력된 경우 오입력 처리 후 새로 재입력한다.

## 6.11 검사기록 목록 화면

### 목적
- 전체 검사기록을 조건별로 조회하고 특정 기록의 세션 상세로 진입한다.

### 주요 구성 요소
- 날짜 범위 필터
- 담당자 필터
- 대상자명 필터
- 오입력 포함 여부 필터
- 기록 목록 테이블

### 기본 정렬
- 최신 검사일시 기준 내림차순

### 목록 단위
- 척도별 기록 단위로 표시한다.
- 한 세션에서 여러 척도를 수행한 경우, 각 척도 기록이 각각 한 줄로 표시된다.

### 목록 기본 표시 항목
- 검사일시
- 대상자명
- 담당자
- 척도명
- 총점
- 판정

### 상세 진입 정책
- 특정 척도 기록 클릭 시 해당 척도가 속한 세션 상세 화면으로 이동한다.
- 세션 상세 화면에서는 클릭한 척도를 기본 강조 표시한다.

### 오입력 표시 정책
- 오입력 기록은 기본 목록에서 숨긴다.
- 작성자와 관리자만 `오입력 포함` 옵션으로 조회 가능하다.

## 8. 상태 및 예외 처리 기준

### 8.1 세션 관련
- 같은 대상자에게 같은 날 여러 세션 생성 가능
- 동일 날짜라도 세션은 생성 시각 기준으로 별도로 구분한다.
- 통계 및 목록에서는 세션 생성 일시 기준으로 구분 가능한 구조로 설계한다.

### 8.2 오입력 관련
- 삭제 기능은 제공하지 않는다.
- 오입력 처리된 기록은 일반 목록에서 숨긴다.
- 오입력 기록은 작성자와 관리자만 조회 가능하다.

### 8.3 중복 등록 관련
- 동일 이름 + 생년월일 등록 시 경고를 제공한다.
- 기존 대상자 후보를 제시하되, 필요 시 신규 등록은 허용한다.

## 9. 개발 시 반영해야 할 UI/UX 메모

- 데스크톱 PC 중심 사용 환경에 맞춰 목록/상세/입력 화면을 넓은 레이아웃으로 설계한다.
- 대상자 목록은 실무형 테이블 UI를 기본으로 한다.
- 척도 입력 화면은 집중 입력에 적합하도록 불필요한 요소를 최소화한다.
- 경고 메시지는 시각적으로 충분히 눈에 띄어야 한다.
- 출력 버튼, 오입력 처리 버튼, 저장 버튼은 성격이 다르므로 시각적으로 구분한다.
- 민감한 상태 변경 버튼은 항상 확인 팝업을 거치도록 한다.

## 11. 결정사항 요약

- 로그인 후 첫 화면은 대상자 목록 화면이다.
- 대상자 목록은 이름, 생년월일, 성별, 사례번호, 담당자, 최근 검사일을 기본 표시한다.
- 새 검사는 대상자 상세 화면에서 시작한다.
- 대상자 상세 화면은 기본정보 + 최근 검사기록 허브 화면으로 구성한다.
- 최근 검사기록은 최근 10건의 세션 단위 요약으로 표시한다.
- 척도 선택은 체크박스 방식으로 복수 선택한다.
- 척도 입력은 한 척도씩 순차 진행한다.
- 척도 진행 순서는 시스템 고정 순서를 따른다.
- 임시저장 기능은 제공하지 않는다.
- 고위험/주의 응답 경고는 즉시 표시한다.
- 세션 전체 요약 화면에서 메모를 입력하고 저장한다.
- 저장 후 세션 상세 화면으로 이동한다.
- 검사기록 목록은 척도별 기록 단위로 조회한다.
- 기록 상세는 세션 전체 상세 화면으로 본다.
- 출력은 세션 상세 화면에서만 제공한다.
- 검사기록은 직접 수정하지 않고, 오입력 처리 후 재입력한다.
- 오입력 처리는 하단 관리 영역에서 작성자/관리자만 수행한다.
- 통계 화면은 전체 현황 요약 탭으로 시작한다.
- 통계 화면은 전 사용자 열람 가능, 엑셀 내보내기는 관리자만 가능하다.
- 세션 참고 메모와 대상자 연락처는 팀 전체 열람 가능하다.
- 대상자 연락처는 선택 입력이다.
- 중복 대상자 등록은 경고 후 허용한다.
- 같은 대상자에게 같은 날 여러 세션 생성이 가능하다.
