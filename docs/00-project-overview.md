# 다시서기 정신건강 평가관리 시스템

## 1. 프로젝트 정보

**프로젝트명**  
다시서기 정신건강 평가관리 시스템

**분야**  
현장 업무경감 자동화 / 기관 내부망 기반 시스템 구축

---

## 2. 프로젝트 목적

다시서기종합지원센터 정신건강팀은 이용자를 대상으로 우울, 불안, 알코올, 수면, 스트레스, 외상, 조울증, 조기정신증 등 다양한 정신건강 척도를 정기적으로 시행하고 있다. 기존에는 종이 기반 또는 수기 방식으로 검사와 채점을 수행한 뒤 별도의 파일에 결과를 다시 입력하는 방식으로 업무가 이루어졌으며, 이 과정에서 채점 오류, 기록 누락, 데이터 분산, 반복 행정업무 등의 문제가 지속적으로 발생하였다.

본 프로젝트는 기관 내부망에서 운영되는 시스템을 통해 8종 정신건강 척도의 시행, 자동 채점, 기록 저장, 조회, 출력, 통계 분석까지 전 과정을 통합 관리하는 것을 목적으로 한다. 이를 통해 실무자의 행정 부담을 줄이고, 데이터의 정확성과 일관성을 높이며, 사례관리 업무의 효율성과 연속성을 강화하고자 한다.

---

## 3. 문제의식 및 필요성

### 1) 반복적 수기 업무로 인한 비효율
정신건강 척도 시행 과정에서 문항 응답 기록, 점수 합산, 결과 판정, 별도 파일 입력까지의 과정이 반복적으로 이루어지고 있어 상당한 시간 소모와 업무 피로를 유발하고 있다.

### 2) 채점 오류 및 기록 누락 위험
PSS-10의 역채점 문항, K-MDQ의 복합 판정 조건 등 구조적으로 오류 가능성이 높은 척도가 포함되어 있어 수기 채점 시 정확성을 안정적으로 보장하기 어렵다.

### 3) 데이터 분산과 활용의 한계
검사 결과가 개별 파일이나 문서에 분산 관리되면 대상자별 변화 추이 확인, 담당자 간 기록 공유, 통계 집계, 보고 자료 작성에 비효율이 발생한다.

### 4) 기관 내부망 환경에 적합한 관리 시스템 필요
정신건강 관련 정보는 민감성이 높은 자료이므로 기관 내부 환경에서 안정적으로 관리할 수 있는 체계가 필요하다. 그러나 현장에서는 다수의 사회복지사가 함께 사용할 수 있는 내부망 기반 평가관리 시스템이 충분히 마련되어 있지 않다.

## 4. 프로젝트 대상 및 운영 환경

### 1) 주요 사용자
- 다시서기종합지원센터 정신건강팀 사회복지사
- 관리자 계정을 가진 운영 담당자

### 2) 운영 환경
- 기관 내부망에서 접속하는 웹 기반 시스템
- 데스크톱 PC 중심 사용 환경
- 다수의 사회복지사가 동시에 접속하여 사용 가능
- 사회복지사가 대상자 응답을 입력하는 실무형 업무 시스템

---

## 5. 프로젝트 내용

### 1) 지원 척도
본 시스템은 아래 8종 정신건강 척도를 통합 지원한다.

- PHQ-9
- GAD-7
- mKPQ-16
- K-MDQ
- PSS-10
- ISI-K
- AUDIT-K
- IES-R

### 2) 대상자 및 검사기록 관리 구조
- 대상자 기본정보와 검사기록을 분리하여 관리
- 대상자 기본정보: 이름, 성별, 생년월일, 사례번호, 연락처, 등록일, 주 담당자
- 사례번호 자동 생성
- 이름 + 생년월일 기반 검색
- 대상자 선택 후 검사 진행
- 필요한 척도를 여러 개 선택하여 한 번에 검사 가능
- 검사 결과는 세션 단위로 저장하고 세션 전체 요약 제공

### 3) 자동 채점 및 판정 기능
- 문항 응답 즉시 총점 자동 산출
- 척도별 판정 구간 자동 표시
- 고위험 또는 주의가 필요한 응답 발생 시 화면 경고 표시
- 전 문항 응답 완료 시에만 저장 가능

### 4) 사용자 권한 및 운영 정책
- 1인 1계정 기반 로그인
- 회원가입 신청 후 관리자 승인 방식
- 관리자 2명 이상 운영 가능
- 관리자 / 일반 사용자 권한 분리
- 팀 전체 열람 가능
- 수정 권한은 작성자와 관리자에게만 부여
- 대상자는 삭제 대신 **오등록 처리(MISREGISTERED)**, 검사 세션/검사기록은 삭제 대신 **오입력 처리(MISENTERED)** 방식을 적용
- 오등록/오입력 상태 데이터는 기본 목록에서 숨기고 관리자 또는 작성자만 확인 가능

### 5) 조회 및 검색 기능
- 대상자 목록 기본 표시: 이름, 생년월일, 성별, 사례번호, 담당자, 최근 검사일
- 검사기록 목록 기본 표시: 검사일, 담당자, 척도명, 총점, 판정
- 필터 조건: 날짜 범위, 담당자, 대상자명, 오입력 제외
- 세션 단위 기록 조회 및 대상자별 이력 확인 가능

### 6) 출력 및 문서화
- 공식 문서형 출력물 제공
- 출력물에 기관명, 팀명, 담당자명 포함
- 대상자 기본정보, 척도별 점수 및 판정, 세션 전체 요약 포함
- 참고 메모는 입력 화면과 상세보기에서만 확인 가능하며 출력물에는 포함하지 않음

### 7) 통계 및 보고 기능
- 시스템 내 통계 화면 제공
- 숫자와 차트 기반으로 주요 현황 확인 가능
- 기본 기간은 이번 주 기준으로 조회
- 척도별 비교 가능
- 경고 기록만 별도로 모아보는 기능 제공
- 엑셀 파일 내보내기를 통해 통계보고 자료로 활용 가능

### 8) 보안 및 운영 안정성
- 2시간 세션 타임아웃 적용
- 주요 기능 중심 로그 기록
- 매일 자동 백업 수행
- 수동 백업 기능 제공
- 백업 파일은 서버 PC의 지정 폴더에 저장

---

## 6. 기대효과

### 1) 행정 업무 효율화
자동 채점과 통합 기록관리 기능을 통해 반복적인 수기 작업을 줄이고 실무자의 행정 부담을 완화할 수 있다.

### 2) 기록 정확성 향상
점수 계산과 판정 과정을 자동화하여 채점 오류와 기록 누락 가능성을 줄일 수 있다.

### 3) 대상자별 기록 누적 및 활용성 강화
검사 이력을 지속적으로 누적하여 대상자별 변화 추이 확인, 사례관리 참고, 내부 보고자료 작성 등에 활용할 수 있다.

### 4) 팀 단위 협업 기반 마련
동일한 내부망 환경에서 여러 사회복지사가 기록을 함께 열람하고 관리할 수 있어 협업 효율이 높아진다.

### 5) 기관 내부 환경에 적합한 운영 지원
기관 내부망 기반 시스템으로 운영함으로써 실무 현장에 맞는 안정적인 기록관리 체계를 구축할 수 있다.

---

## 7. 활용 계획

### 1) 실사용 적용
정신건강팀 사회복지사가 사례관리 대상자의 정기 스크리닝 및 상담 과정에서 본 시스템을 활용하여 검사 시행, 결과 관리, 기록 누적을 일원화한다.

### 2) 운영 및 관리
관리자 계정을 중심으로 사용자 승인, 권한 관리, 로그 확인, 백업 운영을 수행하여 시스템을 안정적으로 운영한다.

### 3) 향후 확장 가능성
향후 척도 추가, 모바일 환경 고려, 통계 기능 고도화 등 단계적 확장이 가능하도록 설계한다.
